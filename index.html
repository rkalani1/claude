Day 0

Set up Claude safely in 20 minutes

The goal is to make Claude useful, predictable, and safe before you feed it important work.

1

Create or confirm your account

Go to claude.ai, sign in, and note whether you are on Free, Pro, Max, Team, or Enterprise. Use Anthropic's current plan chooser before upgrading.

2

Add Instructions for Claude

Open settings and fill in durable preferences: role, tone, recurring formats, and what Claude should ask before assuming.

3

Choose normal or incognito chat

Use normal chats for work you want in your history or memory settings. Use incognito for temporary work; official docs still describe a default 30-day retention period.

4

Run the first useful prompt

Paste the setup interview prompt below. Answer Claude's questions, then ask it to turn the result into your reusable Claude profile.

Core habit

Use the Context - Ask - Inspect - Revise - Save loop

This loop is the beginner version of expert prompting. It works in chat, Projects, Artifacts, Claude Code, and the API.

Context

Tell Claude the job, audience, constraints, sources, and definition of success.

Ask

Ask for one concrete output. Use numbered steps when order matters.

Inspect

Check factual claims, missing assumptions, and whether the answer fits the format.

Revise

Ask Claude to improve against a specific weakness, not just "make it better."

Save

Save the working prompt, Project instruction, Skill, or checklist so you do not start over next time.

First system

Create your first Claude Project

Projects are workspaces with chat histories, knowledge bases, and instructions. Anthropic docs currently say Projects are available to all users, with Free users limited to five Projects.

Beginner setup

  1. Choose one recurring task.
  2. Create a Project named by outcome, not by tool.
  3. Add the Project instruction template from this page.
  4. Upload two or three essential reference files.
  5. Run one test case and improve the instructions.

Expert upgrade

  1. Write a tiny acceptance test.
  2. Add one bad example so Claude learns what to avoid.
  3. Keep standing instructions short.
  4. Prune outdated files monthly.
  5. Use one Project for one workflow.
