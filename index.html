First hour

Start with one real thing you need done

Do not learn Claude by touring menus. Learn it by improving a task you already have.

1

Open Claude

Use Claude in your browser, desktop app, or mobile app. Paste the setup interview prompt and answer one question at a time.

2

Save your preferences

Ask Claude to turn your answers into short Instructions: how you work, preferred tone, formats you like, and what Claude should ask before assuming.

3

Create one Project

Pick one repeated workflow, such as meeting prep, document review, slide creation, coding help, or personal planning. Keep the Project focused.

4

Improve once

After Claude answers, ask it to critique the result and produce a better version. That repeat loop is the fastest path to mastery.

Blank canvas fix

Start from a messy thought

Beginners do not need perfect wording. Give Claude the job, audience, background, and the shape you want back.

Universal task prompt

Difficult email

Messy thought: I need to push back without sounding rude.

Better ask: Draft a firm but warm email to a client explaining why Friday is not realistic, offering two next steps, and keeping the relationship positive.

Long report

Messy thought: I have a huge PDF and need the point.

Better ask: Summarize this report for a busy executive. Start with the main finding, then give five takeaways, risks, and questions I should verify.

Slides from notes

Messy thought: Make these notes into slides.

Better ask: Turn these notes into a six-slide storyline for a mixed audience. For each slide, give the title, one message, suggested visual, and speaker note.

Core habit

Use the Context - Ask - Check - Improve - Save loop

This is the beginner version of expert prompting. It works everywhere Claude works.

Context

Tell Claude the goal, audience, files, constraints, and what good looks like.

Ask

Ask for one concrete output: draft, table, slide outline, code change, summary, or checklist.

Check

Look for missing facts, vague wording, risky assumptions, and wrong format.

Improve

Ask Claude to fix one weakness at a time, then compare versions.

Save

Save the best prompt, Project instruction, Artifact, Office instruction, or code workflow.

Code overview

Chat or Project?

Use Chat for a one-time task. Use a Project when you will repeat the same kind of work, need the same instructions, or want Claude to use several specific documents across conversations.

Quick test

If you would paste the same background again next week, make a Project. If the task ends today, stay in Chat.

Plain English

Claude terms without jargon

Use these short definitions when a menu, announcement, or tutorial uses a term you have not seen before.

Chat

The normal conversation window. Best for quick questions, drafts, summaries, and thinking through a task.

Instructions

Your saved preferences for how Claude should respond: tone, detail level, format, and when to ask questions.

Project

A focused workspace for repeated work that needs the same files, examples, and instructions.

Artifact

A separate editable output Claude can create, such as a document, chart, webpage, prototype, or small app.

Claude Design

A visual creation workflow for pages, prototypes, presentation-style drafts, and polished visual artifacts.

Claude Code

A coding helper that can inspect a repository, explain files, make focused edits, and run checks.

Connector

A way for Claude to work with another app or service when that connection is enabled.

Skill

A reusable set of instructions or files that helps Claude perform a specific kind of work more consistently.

Quick entry

A Mac desktop shortcut for asking Claude about what you are seeing or doing without breaking focus.

Source of truth

The file, note, webpage, or message Claude should treat as the main reference when answers must stay grounded.

Agentic workflow

A repeatable Claude workflow that can gather context, follow steps, use tools, check its work, and hand you a result.

MCP

A standard way to connect Claude Code or agents to tools and data sources with controlled access.

Hook

A command that can run before or after a Claude Code action, often used for checks, formatting, or notifications.

Agent SDK

A developer toolkit for building hosted Claude agents that run inside an app or service.

Tips and tricks

Small phrases that make Claude much better

These are the fastest beginner upgrades. Add one phrase when Claude's answer is close but not good enough.

When the answer is vague

"Make this specific enough that I can act on it today."

When you need quality

"Review your answer against accuracy, clarity, usefulness, and missing assumptions."

When you feel stuck

"Ask me the three questions that would most improve your answer."

When the output is too long

"Give me the shortest useful version, then a fuller version if needed."

When you need options

"Show me three versions: safe, bold, and balanced."

When you want a system

"Turn this into a reusable prompt, checklist, and Project instruction."

When facts matter

"Separate confirmed facts, reasonable inference, and what needs checking."

Answer rescue

Fix Claude when the answer is not good enough

Do not start over. Pick the problem, copy the recovery prompt, and make Claude repair the answer in the same conversation.

What went wrong?

Too vague

Use when: Claude sounds reasonable but you still cannot act on the answer.

Trust check

Before you rely on an answer

Use this when accuracy matters. Claude is strongest when you make it show what is known, what is inferred, and what still needs review.

Ground it

Name the file, note, webpage, or message Claude should treat as the source of truth.

Separate claims

Ask Claude to label confirmed facts, reasonable inferences, and items that need checking.

Check specifics

Review names, dates, numbers, formulas, quotes, links, and anything that could affect a decision.

Ask what changes it

Make Claude say what new information would change the answer or recommendation.

Reliability check prompt

Power moves

Use advanced patterns without jargon

These are the moves that make Claude feel like a working partner instead of a question box.

1

Iterate on Artifacts

After Claude creates an Artifact, ask for changes like adding a filter, turning a memo into a page, adding a toggle, or converting a static draft into an interactive dashboard.

Try: "Keep the same content, but make this easier to scan and add one interactive control."

2

Chain the work

Ask Claude to do several steps in order: extract, sort, compare, flag issues, then create the final output. This reduces back-and-forth.

Try: "First extract dates. Second put them in a table. Third flag conflicts. Fourth write the summary."

3

Make Claude write the prompt

When the task is unclear, ask Claude to interview you before writing the final prompt. This is often faster than guessing what to say.

Try: "Ask me the five questions needed to write the best prompt for this task."

Prompt architect

Agents

Develop and deploy agents with Claude

An agentic workflow is a repeatable job with a goal, inputs, allowed actions, safety checks, and a handoff. Start simple, then automate only after the manual version works.

Beginner

Personal workflow agent

Use a Project, saved prompt, source files, and a checklist. This is best for meeting prep, document review, inbox drafting, research synthesis, and recurring Office work.

Deploy by saving: Project instructions, starter files, a copyable prompt, and a review checklist.

Builder

Repository agent

Use Claude Code when the workflow needs files, tests, command-line checks, or pull requests. Add project rules, custom commands, subagents, hooks, and MCP tools only when they solve a real repeatable problem.

Deploy with: Claude Code, GitHub Actions, or a repo workflow that runs when a task is assigned.

Developer

Hosted product agent

Use the Claude Agent SDK when the agent needs to run inside an app or service. Keep it sandboxed, give it narrow tools, log what it does, and require review before high-impact actions.

Deploy with: a hosted service, container isolation, monitoring, and a clear rollback path.

1

Name the job

Write one sentence: "This agent helps with..." If the job is vague, it is too early to automate.

2

Define the trigger

Say when the workflow starts: a pasted message, a file drop, an issue comment, a daily review, or a manual request.

3

List inputs

Name the files, notes, websites, issues, messages, or databases the agent may use.

4

Set allowed actions

Separate what it may draft, read, edit, click, run, or open from what requires your approval.

5

Add checks

Require source labels, error checks, tests, or a short review checklist before the output is trusted.

6

Ship the smallest version

Try it manually in Chat or a Project, then move to Claude Code, GitHub Actions, or the Agent SDK when the workflow is stable.

Office mastery

Use Claude in Word, PowerPoint, and Excel

Use the Office add-ins where they appear in your Office apps. Keep files open, ask Claude to explain first, and review every change before sharing.

Word

  1. Ask for a plain-English summary.
  2. Ask Claude to find unclear sections.
  3. Edit selected text instead of the whole document.
  4. Use tracked changes when the edit matters.
  5. Ask for a final review checklist.

PowerPoint

  1. Start from your template.
  2. Ask for the storyline before slide creation.
  3. Turn dense bullets into visuals.
  4. Make focused edits to specific slides.
  5. Check brand, formatting, and accuracy before sending.

Excel

  1. Ask Claude to explain the workbook.
  2. Ask what each important formula does.
  3. Request a clean table, chart, or analysis.
  4. Ask for checks that could catch errors.
  5. Use results in a memo or deck after review.

Across Office

  1. Keep the files open.
  2. Tell Claude which file is the source of truth.
  3. Move Excel findings into Word or PowerPoint.
  4. Summarize Word content into slides.
  5. Review changes in each app before saving.
